Why v2.3 Matters

FlowBitz v2.3 is our biggest step forward since launch. Based on real-world Webflow projects, we shipped four headline components, unlocked deeper control over motion behaviour, and trimmed bundle sizes so you can keep Core Web Vitals smiling. This post walks through what’s new and how to start using it on day one.

New Components That Ship Emotion

  • Image Trail — Eight cursor-following presets with GSAP under the hood. Perfect for hero sections and portfolio grids that need personality.
  • 3D Card Hover — A perspective hover effect that feels like a native WebGL experience but runs entirely in CSS.
  • Outline Gradient — Gradient strokes on buttons, cards, and text without adding custom CSS.
  • Roll Text Refresh — Updated presets for vertical reveal choreography powered by SplitText.
<div
  wb-component="image-trail"
  wb-images='["portrait1.webp","portrait2.webp","portrait3.webp"]'
  wb-variant="6"
  wb-trail-gap="12"
></div>

Every component comes with Webflow-friendly attributes, demo content in the playground, and a debugger overlay so you can see thresholds, durations, and trigger states while you iterate.

Smarter Controls for Production Teams

v2.3 adds the kind of knobs agencies kept asking for:

  • Root Margin Options — Use %, vh, px, or any CSS unit to fine-tune scroll triggers.
  • RGBA Color Support — All colour-driven components accept full RGBA strings so you can match brand systems precisely.
  • Hover Toggles — Quickly disable or enable hover states by setting wb-hover="false".
  • Repeat Logic — Decide if animations should replay when elements re-enter the viewport.
<section
  wb-component="smart-animate"
  wb-root-margin="25% 0px"
  wb-repeat="true"
  wb-on-hover="false"
>
  ...
</section>

Performance Wins Without Compromise

Despite the new components, the UMD bundle still weighs in at ~85 KB gzipped, which is 72% smaller than v1. ES modules remain leaner (15 KB entry + lazy chunks). CSS-only variants continue to ship without GSAP to keep marketing pages fast.

  • UMD: includes GSAP for plug-and-play compatibility.
  • ESM: loads GSAP from the CDN only when needed.
  • CSS-only: zero GSAP—ideal for static marketing blocks.

Release Workflow Recap

v2.3 shipped through the same beta → stable pipeline described on the Release Notes page:

  1. Build the library with Vite and publish a beta build via npm run deploy:beta.
  2. Test the beta bundle at https://www.flowbitz.dev/beta/flowbitz.umd.js.
  3. Promote to stable with npm run deploy:promote when it’s ready for production.

Try v2.3 Today

If you’re already using FlowBitz, you can pin the new release via npm or swap the CDN reference:
