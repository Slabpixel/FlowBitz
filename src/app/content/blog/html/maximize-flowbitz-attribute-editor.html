Why Attribute Editor Changes Everything

Webflow's Attribute Editor is the secret weapon that transforms FlowBitz from a powerful library into a creative playground. While you could manually type every wb-* attribute in the code, the Attribute Editor gives you a visual interface to experiment, iterate, and build complex motion systems—all without leaving the Designer. This guide shows you how to unlock that potential.

Getting Started: The Attribute Editor Basics

Every FlowBitz component starts with the wb-component attribute. Here's how to set it up:

  1. Select any element in your Webflow Designer (text, div, button, etc.)
  2. Open the Element Settings panel (or press D)
  3. Scroll to Custom Attributes at the bottom
  4. Click + Add Custom Attribute
  5. Set Name to wb-component
  6. Set Value to your component name (e.g., split-text, image-trail)
<!-- This is what you're creating in Attribute Editor -->
<h1 wb-component="split-text">
  Your animated heading
</h1>

Creative Workflow: Building Component Systems

The real power emerges when you combine multiple attributes to create unique motion experiences. Here are three creative approaches:

1. Layered Animations with Smart Timing

Combine wb-root-margin, wb-stagger-delay, and wb-start-delay to create cascading reveals that feel orchestrated: 

<!-- Hero Section: Text appears before images enter -->
<h1 
  wb-component="split-text"
  wb-split-type="words"
  wb-root-margin="200px 0px"
  wb-stagger-delay="80"
  wb-duration="0.8"
>Welcome to the Future</h1>

<div 
  wb-component="image-trail"
  wb-images='["hero1.webp","hero2.webp"]'
  wb-variant="3"
  wb-root-margin="150px 0px"
></div>

Pro Tip: Use viewport-based units like 25vh for wb-root-margin to create responsive trigger points that adapt to screen height.

2. Dynamic Color Systems with RGBA

FlowBitz v2.3 introduced full RGBA support, which means you can match brand colors precisely and create transparency effects: 

<!-- Gradient text with brand colors and transparency -->
<h2 
  wb-component="gradient-text"
  wb-colors="rgba(64, 255, 170, 1), rgba(64, 121, 255, 0.8), rgba(255, 87, 51, 0.6)"
  wb-animation-speed="6"
>Brand-Aligned Motion</h2>

<!-- Outline gradient with semi-transparent stroke -->
<button 
  wb-component="outline-gradient"
  wb-gradient="linear-gradient(135deg, rgba(64, 255, 170, 0.9), rgba(64, 121, 255, 0.9))"
  wb-border-width="2px"
>Hover Me</button>

Creative Use Case: Use CSS variables in your color attributes to create theme-aware animations that automatically adapt when you switch between light and dark modes in Webflow.

3. Conditional Behaviors with Toggles

Many components support boolean toggles that let you enable or disable behaviors on the fly. This is perfect for A/B testing or responsive design: 

<!-- Disable hover on mobile, enable on desktop -->
<div 
  wb-component="smart-animate"
  wb-on-hover="false"
  wb-trigger-on-view="true"
  wb-repeat="true"
>
  <!-- Content that only animates on scroll, not hover -->
</div>

<!-- Magnetic button that only works on desktop -->
<button 
  wb-component="magnetic-button"
  wb-hover="true"
  wb-strength="50"
>Interactive Element</button>

Advanced Techniques: Pushing the Boundaries

Multi-Text Sequences

The text-type component supports unlimited text strings via wb-text-1, wb-text-2, wb-text-3, and beyond. Create dynamic hero sections that cycle through value propositions: 

<span 
  wb-component="text-type"
  wb-text-1="Build faster"
  wb-text-2="Ship smarter"
  wb-text-3="Scale effortlessly"
  wb-text-4="Deliver excellence"
  wb-typing-speed="80"
  wb-pause-duration="2500"
  wb-cursor-character="pipe"
  wb-show-cursor="true"
></span>

Image Trail Variants

The image-trail component offers eight preset variants, each with unique motion characteristics. Experiment with different variants to match your brand's personality:

  • Variant 1-2: Subtle, elegant follows—perfect for professional portfolios
  • Variant 3-4: Dynamic, energetic motion—ideal for creative agencies
  • Variant 5-6: Playful, bouncy interactions—great for consumer brands
  • Variant 7-8: Advanced physics-based effects—for cutting-edge projects
<div 
  wb-component="image-trail"
  wb-images='["team1.webp","team2.webp","team3.webp","team4.webp"]'
  wb-variant="6"
  wb-trail-gap="15"
  wb-image-width="200px"
  wb-image-height="250px"
></div>

Precision Scroll Control

The wb-root-margin attribute accepts any CSS unit, giving you pixel-perfect control over when animations trigger:

  • "200px 0px" — Triggers 200px before element enters viewport (early reveal)
  • "0px" — Triggers exactly when element enters (standard)
  • "-100px 0px" — Triggers 100px after element enters (late reveal)
  • "25vh 0px" — Viewport-relative trigger (responsive)
  • "10% 0px" — Percentage-based trigger (scales with element size)

Workflow Optimization: Speed Up Your Process

Create Reusable Component Classes

Instead of adding attributes to every element, create Webflow classes with preset attribute configurations:

  1. Add attributes to one element and perfect the settings
  2. Create a class from that element (e.g., .hero-split-text)
  3. Apply the class to other elements—attributes copy automatically
  4. Update the class to change all instances at once

Pro Tip: Use Webflow's Combo Classes to create variations (e.g., .hero-split-text.fast with wb-stagger-delay="50").

Use Symbols for Complex Components

For multi-element components like image-trail or nested animations, convert them to Webflow Symbols:

  • Build the complete component with all attributes configured
  • Select the parent container and create a Symbol
  • Instance the Symbol across pages—all attributes stay intact
  • Edit the master Symbol to update all instances globally

Leverage the Debugger Overlay

FlowBitz includes a built-in debugger that shows trigger states, timelines, and thresholds. Enable it by adding ?debug=true to your preview URL or setting wb-debug="true" on any component. Use it to:

  • Verify wb-root-margin values are triggering at the right scroll position
  • Check if wb-repeat is working as expected
  • See animation durations and delays in real-time
  • Identify performance bottlenecks during development

Common Pitfalls and How to Avoid Them

  • Attribute Name Typos: Always double-check attribute names. wb-stagger-delay (correct) vs wb-staggerdelay (won't work). Use the component documentation as reference.
  • JSON Array Formatting: For attributes like wb-images, ensure proper JSON formatting with quotes around strings: '["img1.webp","img2.webp"]'
  • Unit Consistency: Mixing units in wb-root-margin can cause unexpected behavior. Stick to one unit type per value (e.g., all px or all vh).
  • Over-animating: Too many simultaneous animations can hurt performance. Limit to 2-3 components per viewport and use wb-disabled to toggle components during testing.

  5. Share Your Work: Tag @flowbitz when you ship something amazing
