FlowBitz at a Glance

FlowBitz is a motion component library designed for Webflow teams who want GSAP-level polish without leaving the visual Designer. Drop in a single script, configure components with wb-* attributes, and FlowBitz takes care of timeline orchestration, lifecycle management, and performance tuning.

  • 20+ production-ready text, button, and effects components
  • Attribute-based configuration—no custom JavaScript required
  • GSAP-powered with CSS fallbacks for lean, resilient builds
  • Debugger overlays, metadata, and presets built for fast iteration
<!-- One script unlocks the full FlowBitz library -->
<script src="https://cdn.jsdelivr.net/npm/flowbitz@latest/dist/flowbitz.umd.js"></script>

<h1 wb-component="split-text" wb-split-type="words">
  Explain your value prop with motion that converts
</h1>

Why Webflow Developers Love It

FlowBitz was born inside a Webflow studio that needed to ship high-end motion without breaking no-code workflows. Every component is mapped to Webflow’s structure and naming conventions so designers can:

  • Clone components across pages without re-writing logic
  • Change animation behaviour by editing attributes right in the Designer
  • Debug timelines via the FlowBitz overlay while previewing the site
  • Switch between beta/stable builds with scripted deploy commands
<div
  wb-component="image-trail"
  wb-images='["portrait1.webp","portrait2.webp","portrait3.webp"]'
  wb-variant="6"
></div>

Built for Webflow—Ready for Anything Else

While FlowBitz feels native to Webflow, it’s ultimately just JavaScript. If you outgrow Webflow or need to embed components elsewhere, the same bundle works in:

  • React, Next.js, and other frameworks via the ES module build
  • Static sites by importing the UMD bundle
  • Legacy projects by loading the CDN script and initialising components manually
import FlowBitz from 'flowbitz'

await FlowBitz.init({
  components: ['splitText', 'magneticButton'],
  debug: false
})

// Programmatic control is available everywhere
const splitText = FlowBitz.getComponent('splitText')
splitText.replay(document.querySelector('[wb-component="split-text"]'))

Deployment Workflow You Can Trust

We treat FlowBitz like a product, not a side project. Every update runs through a beta channel before promotion to stable:

  1. Develop in Webflow using the local Vite playground
  2. Release a beta build with npm run deploy:beta and test via the CDN
  3. Promote to stable when everything checks out—your production sites stay locked to the latest stable automatically
